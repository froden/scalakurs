Lister

I denne oppgaven skal vi jobbe med Scalas meget kraftige APIer for å håndtere lister. scala.immutable.List er i motsetning til List i Java immutable - den kan ikke endres. Det vil si at alle metoder som endrer en liste, returnerer en ny liste med endringene. Metodene på List sparer oss for mye tid og potensielle feil med å implementere traverseringslogikken for oss. Disse metodene tar som regel en funksjon som parameter. Se seksjonen nedenfor for syntaks for funksjoner i Scala.

Flere metoder i liste-api tar inn funksjoner som argument. Under er count-metoden brukt. Den tar inn en funksjon som parameter. En egendefinert metode som teller partall i en liste med Int ser da slik ut:

def countEvenNumbers (myList: List[Int]) = myList.count((n: Int) => n % 2 == 0)

Funksjoner

Funksjoner skiller seg fra metoder ved at de kan tilordnes til variabler og har en egen type:

val myAdd = (n1: Int, n2: Int) => n1 + n2

En tilsvarende metode ville sett ut som følger:

def myAdd2(n1: Int, n2: Int) = n1 + n2

myAdd over har typen (Int, Int) => Int altså "en funksjon som tar 2 Int som parameter og returnerer Int". Hvis vi deklarerer typen til myAdd (valgfritt) vil vi få følgende syntaks:

val myAdd: (Int, Int) => Int = (n1: Int, n2: Int) => n1 + n2

Andre gyldige kortformer som er nyttige når man sender funksjoner som parametre til andre fuksjoner/metoder:

val myAdd: (Int, Int) => Int = (n1, n2) => n1 + n2
val myAdd: (Int, Int) => Int = _ + _
